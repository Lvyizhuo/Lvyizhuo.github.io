## 16:55 左右：修复页面顶部乱码
- 问题：用户发现页面顶部出现一大段重复乱码，形如 `{# Emit a root-relative base path ... #}`。
- 根因：`_includes/base_path` 里用了 `{# ... #}` 作为注释，但这不是合法 Liquid 语法，Liquid 将其当作普通文本输出。该文件被导航链接多次 include，导致乱码重复出现。
- 修复：把 `_includes/base_path` 里的 `{# ... #}` 替换为合法 `{% comment %}...{% endcomment %}` 注释；文件只保留赋值语句，不再输出多余文本。
- 验证：Docker 实时页面中「Emit a root-relative」计数为 0，导航/侧栏/正文正常，CSS 200，无头 Chrome 截图无乱码。

## 21:55+ 修复：Projects 卡片深色模式不跟随切换
- 现象：切换深色模式时卡片仍是白底、观感没切。
- 根因：站点通过 JS 给 `<html>` 加 `data-theme="dark"` 切换（见 `_sass/theme/_dark.scss` 变量 + `_main.js` toggle）；但 `.paper-box` 背景写死 `#ffffff`，无深色覆盖；文字色靠主题变量会切，却被白底盖住。
- 修复（`assets/css/custom.css`）：
  - `.paper-box-text` 显式 `color: var(--global-text-color)`；`.paper-box-text a` 显式 `color: var(--global-link-color)`，文字/链接跟随主题变量。
  - 新增 `html[data-theme="dark"] .paper-box` 等深色规则（共 4 条，含 `.badge`）：卡片表面 `#1f2937`、边框/图片边框 `var(--global-border-color)`、hover 阴影加深；`.badge` 深色下文字改 `#0b1220`（蓝底白字对比太低）。
  - 注意 grep 选择器是 `html[data-theme="dark"] .paper-box`（带 `]`），漏 `]` 会匹配不到。
- 验证：`jekyll build` 通过；`_site` 与 localhost:4000 的 custom.css 均含深色规则；页面仍正常渲染卡片。
