## 16:55 左右：修复页面顶部乱码
- 问题：用户发现页面顶部出现一大段重复乱码，形如 `{# Emit a root-relative base path ... #}`。
- 根因：`_includes/base_path` 里用了 `{# ... #}` 作为注释，但这不是合法 Liquid 语法，Liquid 将其当作普通文本输出。该文件被导航链接多次 include，导致乱码重复出现。
- 修复：把 `_includes/base_path` 里的 `{# ... #}` 替换为合法 `{% comment %}...{% endcomment %}` 注释；文件只保留赋值语句，不再输出多余文本。
- 验证：Docker 实时页面中「Emit a root-relative」计数为 0，导航/侧栏/正文正常，CSS 200，无头 Chrome 截图无乱码。
